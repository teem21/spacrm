# STEP-05: Комбо-пакеты CRUD

## Зависимости

- **STEP-01**: Storage layer, тип ComboPackage.
- **STEP-04**: Список процедур (нужен для выбора шагов комбо).

## Scope

Подвкладка "Комбо-пакеты" в экране "Услуги и цены". CRUD для комбо-пакетов.

## Что на экране

**Таблица комбо:**
- Столбцы: Название | Состав (перечень шагов через →) | Общее время | Цена | Активен
- Данные из `spa-crm:combos:{activeSalonId}`.

**Создание (кнопка "+ Создать комбо"):**
- Модалка или раскрывающаяся панель:
  - Название (text, обязательно).
  - Шаги: список с возможностью добавить процедуру из dropdown активных процедур.
    - Каждый шаг: название процедуры + длительность + кнопки ↑↓ для смены порядка + кнопка ✕ для удаления шага.
    - Кнопка "+ Добавить шаг".
  - Общее время: авто-расчёт (сумма длительностей). Read-only.
  - Цена: number input. По умолчанию = сумма цен процедур. Юзер может задать скидочную цену.
  - Кнопки: Сохранить / Отмена.

**Правила:**
- Минимум 2 шага в комбо.
- Сауна может быть только первым шагом (если добавлена — автоматически ставится на позицию 1).
- Пиллинг может быть только сразу после сауны (позиция 2, и только если шаг 1 = сауна).
- Нельзя добавить пиллинг без сауны.

**Редактирование:** клик по строке → та же модалка/панель с заполненными данными.

**Деактивация:** toggle, аналогично процедурам.

## UI

- Модалка создания: ширина 520px, фон #1A2332, overlay.
- Шаги в модалке: вертикальный список с визуальными стрелками → между ними.
- Авто-расчёт обновляется при каждом изменении шагов.

## Storage

- `spa-crm:combos:{salonId}` — JSON массив ComboPackage.
