# SPA CRM — Полная спецификация v4

## Обзор проекта

React-артефакт (single-file JSX) для управления двумя спа-салонами тайского массажа. Все данные хранятся в persistent storage (`window.storage`). Язык интерфейса — русский.

**Технический стек:** React (hooks), Tailwind CSS (core utilities), Recharts, Lucide-react, lodash. Single-file JSX. Нет form-тегов — только onClick/onChange. Нет localStorage/sessionStorage.

## 1. Сущности и данные

### 1.1 Салон (Salon)

Фиксированно 2 салона. Каждый имеет:

- `id`: "salon-1" | "salon-2"
- `name`: string — название (задаёт юзер)
- `rooms`: массив кабинок
- `therapistCount`: number — количество таек (мастеров)
- `hasSauna`: boolean — есть ли сауна
- `saunaCapacity`: number — макс кол-во людей в сауне одновременно
- `hasPeeling`: boolean — есть ли зона пиллинга
- `peelingMaxPerHour`: number — макс кол-во людей на пиллинг за 1 час сауны
- `peelingMastersMax`: number — макс кол-во мастеров на пиллинг одновременно
- `peelingTimePerPerson`: number — минут на пиллинг 1 человека (по умолчанию 30)
- `saunaDuration`: number — длительность сауны в минутах (по умолчанию 60)
- `workStart`: "10:00" — начало рабочего дня
- `workEnd`: "22:00" — конец рабочего дня
- `dayOff`: "monday" — выходной день
- `bufferMinutes`: number (по умолчанию 15) — буфер между записями

### 1.2 Кабинка (Room)

- `id`: string
- `name`: string ("Кабинка 1", "Кабинка 2"…)
- `beds`: 1 | 2 — количество кроватей

### 1.3 Процедура (Procedure)

Динамический справочник — юзер создаёт/редактирует/удаляет процедуры для каждого салона.

- `id`: string (uuid)
- `salonId`: string
- `name`: string
- `category`: "massage" | "sauna" | "peeling"
- `duration`: number — длительность в минутах
- `price`: number — цена
- `therapistsRequired`: number — сколько мастеров нужно на 1 клиента (по умолчанию 1; для "массажа в 4 руки" = 2)
- `isActive`: boolean

**Процедуры по умолчанию:**

| Название             | Категория | Длительность | Мастеров/клиент |
|----------------------|-----------|--------------|-----------------|
| Тайский массаж 1ч   | massage   | 60 мин       | 1               |
| Тайский массаж 1.5ч | massage   | 90 мин       | 1               |
| Тайский массаж 2ч   | massage   | 120 мин      | 1               |
| Ойл массаж 1ч       | massage   | 60 мин       | 1               |
| Ойл массаж 1.5ч     | massage   | 90 мин       | 1               |
| Ойл массаж 2ч       | massage   | 120 мин      | 1               |
| Массаж в 4 руки 1ч  | massage   | 60 мин       | 2               |
| Массаж в 4 руки 1.5ч| massage   | 90 мин       | 2               |
| Массаж в 4 руки 2ч  | massage   | 120 мин      | 2               |
| Сауна               | sauna     | 60 мин       | 0               |
| Пиллинг             | peeling   | 30 мин       | 1               |

### 1.4 Комбо-пакет (ComboPackage)

- `id`: string (uuid)
- `salonId`: string
- `name`: string
- `steps`: массив шагов `{ procedureId, order }`
- `totalDuration`: number — авто-расчёт сумма длительностей
- `price`: number — цена пакета (юзер может задать скидку)
- `isActive`: boolean

**Логика комбо при записи:**

- Шаги выполняются СТРОГО последовательно.
- Между шагами буфер НЕ добавляется. Буфер только в конце всего комбо.
- Сауна всегда первой, если присутствует.
- Пиллинг параллельно с сауной (внутри её часа).
- После сауны клиент переходит в кабинку → бронирование кабинки с момента окончания сауны.

### 1.5 Запись/Бронирование (Booking)

- `id`: string (uuid)
- `salonId`: string
- `date`: string (YYYY-MM-DD)
- `clientName`: string
- `clientPhone`: string
- `clientCount`: number
- `bookingType`: "single_procedure" | "combo"
- `procedureId`: string | null
- `comboId`: string | null
- `segments`: массив сегментов:
  - `procedureId`, `procedureName`, `startTime` (HH:MM), `endTime` (HH:MM), `roomId` (null для сауны), `therapistCount`, `resourceType` ("room" | "sauna" | "peeling")
- `totalStartTime`: string (HH:MM)
- `totalEndTime`: string (HH:MM) — включая буфер
- `totalPrice`: number
- `status`: "booked" | "completed" | "cancelled" | "no-show"
- `createdAt`: string (ISO)
- `notes`: string

### 1.6 Лог клиентов (ClientLog)

Автоматически формируется из записей. Не хранится отдельно — строится из bookings.

## 2. Бизнес-логика

### 2.1 Правила записи

**Кабинки:**
- 2 кровати → принимает пару (2 человека вместе). Два случайных человека НЕ в одну кабинку.
- 1 кровать → только 1 человек.
- 3+ человек → несколько кабинок (авто-распределение: пары в 2-кроватные, одиночки в любую).

**Мастера:**
- Обычная процедура: 1 мастер на 1 клиента.
- Массаж в 4 руки: 2 мастера на 1 клиента.
- Формула: `clientCount × procedure.therapistsRequired`.
- Мастера на пиллинг из общего пула.

**Сауна:**
- 1 сауна на салон. Одновременно только 1 группа.
- Вместимость и длительность задаются юзером.

**Пиллинг:**
- Внутри сеанса сауны, параллельно.
- Формула: `peelingMaxPerHour ≤ (saunaDuration / peelingTimePerPerson) × peelingMastersMax`.
- Мастера из общего пула.

**Буфер:**
- После каждой записи/комбо. Между шагами комбо — НЕ добавляется.

### 2.2 Расчёт времени

```
Одиночная: startTime + duration + buffer
Комбо:     step1.start → +step1.duration → step2.start → +step2.duration → ... → lastEnd + buffer
С пиллингом: сауна (пиллинг внутри параллельно) → saunaEnd → massage → lastEnd + buffer
```

### 2.3 Валидация

1. Дата не выходной.
2. Вся запись + буфер ≤ workEnd.
3. Ресурс (кабинка/сауна) свободен.
4. Достаточно мастеров (`clientCount × therapistsRequired` + пиллинг).
5. Сауна свободна, клиенты ≤ saunaCapacity.
6. Пиллинг ≤ peelingMaxPerHour, хватает мастеров.
7. Пара → кабинка с 2 кроватями.
8. 3+ → достаточно кабинок.

## 3. Persistent Storage

### Ключи:

```
spa-crm:salons                      → конфигурация обоих салонов
spa-crm:procedures:salon-1           → процедуры салона 1
spa-crm:procedures:salon-2           → процедуры салона 2
spa-crm:combos:salon-1               → комбо салона 1
spa-crm:combos:salon-2               → комбо салона 2
spa-crm:bookings:salon-1:YYYY-MM     → записи салона 1 за месяц
spa-crm:bookings:salon-2:YYYY-MM     → записи салона 2 за месяц
```

### Storage API:

```javascript
await window.storage.get(key)       → {key, value} | throws
await window.storage.set(key, value) → {key, value}
await window.storage.delete(key)     → {key, deleted}
await window.storage.list(prefix)    → {keys}
```

Все операции — try/catch. Несуществующие ключи бросают ошибку.

- [ ] Два салона конфигурируемы (кабинки, мастера, часы, буфер).

- [ ] Запись с полной валидацией ресурсов.

- [ ] Индикатор мастеров под сеткой.
