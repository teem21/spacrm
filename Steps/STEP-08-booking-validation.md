# STEP-08: Валидация записей

## Зависимости

- **STEP-01**: Storage layer, данные салонов.
- **STEP-07**: Форма записи (валидация вызывается при нажатии "Записать").

## Scope

Полная бизнес-валидация при создании записи. Все проверки выполняются ДО сохранения.

## Правила валидации

### 1. Дата и время

- Дата НЕ является выходным днём (`salon.dayOff`).
- Время начала ≥ workStart.
- Время окончания последнего сегмента + буфер ≤ workEnd.

### 2. Кабинки

Для каждого сегмента с resourceType = "room":
- Кабинка свободна на весь интервал [startTime, endTime].
- Проверка: ни один другой сегмент другой записи на эту дату не пересекается по кабинке и времени.

```javascript
function isRoomFree(roomId, start, end, existingBookings, excludeBookingId) {
  for (const booking of existingBookings) {
    if (booking.id === excludeBookingId) continue;
    for (const seg of booking.segments) {
      if (seg.roomId === roomId && seg.startTime < end && seg.endTime > start) {
        return false;
      }
    }
  }
  return true;
}
```

### 3. Сауна

Если сегмент с resourceType = "sauna":
- Сауна свободна на интервал (ни одна другая запись не занимает сауну).
- clientCount ≤ saunaCapacity.

### 4. Мастера

Для каждого 15-минутного слота, покрываемого новой записью:
- Суммарное кол-во мастеров (существующие + новая) ≤ salon.therapistCount.
- Учитывать: therapistCount из каждого сегмента, пересекающегося со слотом.
- Для пиллинга: мастера на пиллинг учитываются в слотах, где пиллинг происходит (внутри сауны).

```javascript
function getTherapistLoad(bookings, slotStart, slotEnd) {
  let load = 0;
  for (const booking of bookings) {
    for (const seg of booking.segments) {
      if (seg.startTime < slotEnd && seg.endTime > slotStart) {
        load += seg.therapistCount;
      }
    }
  }
  return load;
}
```

### 5. Пиллинг

- Кол-во людей на пиллинг ≤ peelingMaxPerHour.
- Хватает мастеров: проверка через п.4 (мастера на пиллинг добавляются в therapistCount сегмента пиллинга).

### 6. Кабинки для пар/компаний

- clientCount = 2: назначенная кабинка должна иметь beds = 2.
- clientCount = 3+: все назначенные кабинки свободны, суммарно beds покрывают всех.

### 7. Массаж в 4 руки

- Процедура с therapistsRequired = 2: `requiredTherapists = clientCount × 2`.
- Проверка через п.4.

## Формат ошибок

Функция `validateBooking(booking, existingBookings, salon)` возвращает:

```javascript
{
  valid: boolean,
  errors: [
    { field: 'date', message: 'Выбран выходной день (Понедельник)' },
    { field: 'time', message: 'Запись выходит за рабочие часы (до 22:00)' },
    { field: 'room', message: 'Кабинка 1 занята с 10:30 до 11:30' },
    { field: 'therapists', message: 'Недостаточно мастеров (нужно 4, свободно 2 в 10:30)' },
    { field: 'sauna', message: 'Сауна занята с 10:00 до 11:00' },
  ]
}
```

## Отображение ошибок

- Каждая ошибка — красный текст (#F87171) под соответствующим полем.
- Если несколько ошибок — показать все.
- Кнопка "Записать" заблокирована (disabled) при наличии ошибок.
- Валидация запускается при нажатии "Записать" и при изменении ключевых полей (время, кабинка, кол-во клиентов).

## Acceptance Criteria

- [ ] Запись на выходной — ошибка.
- [ ] Запись за пределами рабочих часов — ошибка.
- [ ] Конфликт кабинки — ошибка с указанием времени конфликта.
- [ ] Недостаточно мастеров — ошибка с числами.
- [ ] Сауна занята — ошибка.
- [ ] Пара в кабинке с 1 кроватью — ошибка.
- [ ] Массаж в 4 руки: 2 клиента = 4 мастера проверяется.
- [ ] Ошибки показываются пользователю в модалке.
- [ ] Кнопка "Записать" не работает при наличии ошибок.
- [ ] При исправлении — ошибки исчезают.
