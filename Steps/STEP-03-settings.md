# STEP-03: Настройки салонов

## Зависимости

- **STEP-01**: Storage layer, салон state (salons), app shell с табом "Настройки".

## Scope

Экран настроек: редактирование конфигурации обоих салонов. Изменения сохраняются в `spa-crm:salons`.

## Что на экране

Два карточных блока (Салон 1 и Салон 2), вертикально. Каждый блок:

- **Название салона** — text input.
- **Кол-во кабинок** — number input (1-6). При изменении числа: динамически добавить/удалить строки кабинок.
- **Список кабинок** — для каждой: название (text) + кол-во кроватей (1 или 2, toggle или select).
- **Кол-во мастеров** — number input (1-15).
- **Рабочие часы** — два time-select: начало (workStart) и конец (workEnd). Шаг 30 мин.
- **Выходной день** — dropdown (пн-вс).
- **Буфер** — number input (5-60 мин, шаг 5).

Внизу страницы: раздел "Данные" (заглушка для STEP-12).

## UI

```
┌─── Салон 1: Thai Orchid ──────────────────────────┐
│ Название   [ Thai Orchid              ]           │
│ Кабинок    [ 3 ]                                  │
│  • Кабинка 1 — [ 2 ] кровати                      │
│  • Кабинка 2 — [ 1 ] кровать                      │
│  • Кабинка 3 — [ 2 ] кровати                      │
│ Мастеров   [ 6 ]                                  │
│ Часы       [ 10:00 ] — [ 22:00 ]                  │
│ Выходной   [ Понедельник ▾]                       │
│ Буфер      [ 15 ] мин                             │
└───────────────────────────────────────────────────┘
```

- Карточки: фон #1A2332, border-radius 12px, padding 24px.
- Auto-save: изменения сохраняются в storage при каждом blur/change (debounced 500ms).
- Toast "Сохранено" при успешном сохранении.

## Acceptance Criteria

- [ ] Таб "Настройки" показывает конфигурацию обоих салонов.
- [ ] Все поля редактируемы.
- [ ] Изменение кол-ва кабинок динамически добавляет/удаляет строки.
- [ ] Данные auto-save в storage.
- [ ] Toast-уведомление при сохранении.
- [ ] Валидация: кабинки 1-6, мастера 1-15, буфер 5-60, workStart < workEnd.
