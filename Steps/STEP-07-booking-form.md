# STEP-07: Форма записи (создание бронирования)

## Зависимости

- **STEP-01**: Storage layer, тип Booking, uuid генерация.
- **STEP-04**: Список процедур (для выбора).
- **STEP-05**: Список комбо-пакетов (для выбора).
- **STEP-06**: Сетка расписания (форма открывается из кнопки "+ Новая запись" и при клике на пустой слот).

## Scope

Модальное окно создания новой записи. Сбор данных, генерация сегментов, базовая валидация (полная валидация — STEP-08).

## Модалка

```
┌──────────────── Новая запись ─────────────────────┐
│                                              [✕]  │
│  Имя клиента     [ __________________________ ]   │
│  Телефон         [ __________________________ ]   │
│  Кол-во клиентов [ 1 ▾]                           │
│                                                   │
│  (●) Одна процедура  ( ) Комбо-пакет              │
│  Процедура       [ Тайский массаж 1ч  ▾]          │
│                                                   │
│  Дата            [ 16.03.2026 ]                   │

│                                                   │

│                                                   │

```

**Размер:** 520px ширина, центрирована, overlay чёрный 60% opacity.
**Фон:** #1A2332, border-radius 12px, padding 24px.

## Поля и логика

### 1. Имя клиента
- Text input, обязательно.

### 2. Телефон
- Text input, обязательно.

### 3. Кол-во клиентов
- Dropdown: 1-4 (если процедура НЕ включает сауну), 1-saunaCapacity (если включает сауну).

### 4. Тип записи
- Radio: "Одна процедура" / "Комбо-пакет".

### 5. Выбор процедуры (если "Одна процедура")
- Dropdown из активных процедур текущего салона.
- При выборе massage с therapistsRequired = 2 → предупреждение "⚠ Требуется 2 мастера на клиента".

### 6. Выбор комбо (если "Комбо-пакет")
- Dropdown из активных комбо текущего салона.
- При выборе — показать timeline: `Сауна (10:30–11:30) → Тайский (11:30–12:30)`.
- Если комбо содержит пиллинг → дополнительное поле: "Кол-во человек на пиллинг" (1 до peelingMaxPerHour).

### 7. Дата
- Предзаполнена текущей выбранной датой из расписания. Можно изменить.

### 8. Время начала
- Dropdown, шаг 15 мин, от workStart до workEnd минус длительность.
- При клике на пустой слот в сетке — предзаполнено временем слота.

### 9. Кабинка
- Auto-suggest: первая свободная кабинка подходящего типа.
- Для clientCount = 2: только кабинки с beds = 2.
- Dropdown для ручного выбора.
- Для clientCount = 3+: авто-распределение (пары в 2-кроватные, одиночки в любые). Показывать результат: "Кабинка 1 (2 чел.) + Кабинка 3 (1 чел.)".

### 10. Итого (read-only блок)
- Длительность: расчёт (процедура или сумма комбо + буфер).
- Мастеров: clientCount × therapistsRequired (+ пиллинг если есть).
- Цена: procedure.price × clientCount (или combo.price × clientCount).

### 11. Заметки
- Textarea, опционально.

## Генерация сегментов

### Одиночная процедура:

```javascript
segments = [{
  procedureId, procedureName, 
  startTime, endTime: startTime + duration,
  roomId, therapistCount: clientCount * therapistsRequired,
  resourceType: procedure.category === 'sauna' ? 'sauna' : 'room'
}];
totalEndTime = endTime + buffer;
```

### Комбо:

```javascript
let currentTime = startTime;
segments = [];
for (const step of combo.steps) {
  const proc = procedures.find(p => p.id === step.procedureId);
  const endTime = currentTime + proc.duration;
  segments.push({
    procedureId: proc.id, procedureName: proc.name,
    startTime: currentTime, endTime,
    roomId: proc.category === 'sauna' ? null : assignedRoomId,
    therapistCount: proc.category === 'sauna' ? 0 : clientCount * proc.therapistsRequired,
    resourceType: proc.category
  });
  currentTime = endTime; // НЕТ буфера между шагами
}
totalEndTime = currentTime + buffer; // буфер только в конце
```

### Пиллинг (внутри сауны):
Если комбо содержит пиллинг после сауны — пиллинг-сегмент НЕ добавляет отдельное время. Он параллелен сауне. Но его therapistCount учитывается в слотах пересечения.

```javascript
// Пиллинг: startTime = sauna.startTime, endTime = sauna.startTime + peelingCount * peelingTimePerPerson
// resourceType = 'peeling', roomId = null
// therapistCount = min(peelingCount, peelingMastersMax) — одновременно работают не более peelingMastersMax мастеров
```

## Сохранение

При нажатии "Записать":
1. Сгенерировать segments.
2. Провести валидацию (STEP-08 расширит).
3. Если ok → создать объект Booking, добавить в `spa-crm:bookings:{salonId}:{YYYY-MM}`.
4. Закрыть модалку, обновить сетку.
5. Toast "Запись создана".
