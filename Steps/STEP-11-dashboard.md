# STEP-11: Дашборд — KPI, графики, сравнение

## Зависимости

- **STEP-01**: Storage layer, данные салонов.
- **STEP-10**: Загрузка всех bookings (тот же механизм).

## Scope

Таб "Дашборд" — аналитика с KPI-карточками, графиками и сравнением салонов. Используем Recharts.

## UI Layout

```
┌──────────────────────────────────────────────────────────┐
│  [Салон 1][Салон 2][Оба]     [Сегодня|Неделя|Месяц]      │
├──────────────────────────────────────────────────────────┤
│  [42 записей] [58 клиентов] [290K выручка] [5000 ср.чек]│
│  [78% кабинки] [65% мастера]                             │
├──────────────────────────────────────────────────────────┤
│  [Bar: Выручка/дни]         [Pie: Типы процедур]         │
│  [Bar: Загрузка/часы]       [HBar: Топ процедур]         │
└──────────────────────────────────────────────────────────┘
```

### Переключатели:

**Салон:** Салон 1 / Салон 2 / Оба — pill-tabs, стиль как переключатель салонов.
**Период:** Сегодня / Неделя / Месяц — pill-tabs. "Сегодня" = текущая дата. "Неделя" = последние 7 дней. "Месяц" = текущий месяц.

### KPI-карточки (6 шт):

Расчёт по выбранному салону/периоду. Учитывать только записи со статусом "booked" или "completed" (НЕ cancelled, НЕ no-show).

| KPI               | Расчёт                                                      |
|-------------------|--------------------------------------------------------------|
| Всего записей     | count(bookings в периоде)                                    |
| Всего клиентов    | sum(booking.clientCount)                                     |
| Выручка           | sum(booking.totalPrice) для completed                        |
| Средний чек       | выручка / кол-во записей completed                           |
| Загрузка кабинок  | (занятые минуты кабинок / общие доступные минуты) × 100%     |
| Загрузка мастеров | (занятые мастеро-минуты / общие мастеро-минуты) × 100%       |

**Визуал карточки:**
- Число крупно (24px, вес 700, цвет #E8E0D6).
- Подпись мелко (11px, #8A9AAE).
- Для загрузки: процент + CSS ring (мини-кольцевая диаграмма).
- Grid: 4 в первом ряду + 2 во втором.
- Фон карточки: #1A2332, border-radius 8px, padding 16px.

### Графики (4 шт, grid 2×2):

**1. Выручка по дням (BarChart):**
- X: даты периода. Y: сумма totalPrice (completed).
- Цвет: #D4A84B.
- Для "Оба" — два набора (stacked или grouped).

**2. Распределение по типам процедур (PieChart):**
- Сегменты: massage / sauna / peeling / combo.
- Цвета: #2D6A4F, #B85C38, #7B68AE, #D4A84B.

**3. Загрузка по часам (BarChart):**
- X: часы (10:00, 11:00, ..., 21:00).
- Y: среднее кол-во занятых мастеров в этот час за период.
- Цвет: #2D6A4F.

**4. Топ процедур (Horizontal BarChart):**
- Топ-5 процедур по количеству записей.
- Горизонтальные бары, названия слева.
- Цвет: #D4A84B.

**Стиль графиков:**
- Контейнер: #1A2332, border-radius 8px, padding 16px.
- Заголовок графика: 13px, вес 600.
- Recharts: тёмные оси (#2A3A4E), текст осей #8A9AAE, grid-lines #2A3A4E.
- Tooltip: фон #1A2332, border #2A3A4E.

### Сравнение салонов (режим "Оба"):

Дополнительный блок внизу: таблица метрик бок о бок.

| Метрика      | Салон 1 | Салон 2 |
|-------------|---------|---------|
| Записей     | 42      | 35      |
| Выручка     | 290K ₸  | 210K ₸  |
| Ср. чек     | 5 000 ₸ | 4 800 ₸ |
| Загр. каб.  | 78%     | 65%     |
| Загр. маст. | 65%     | 58%     |

Подсвечивать лучший показатель золотым.

## Acceptance Criteria

- [ ] Переключатель салонов и периодов работает.
- [ ] 6 KPI-карточек с корректными расчётами.
- [ ] Загрузка кабинок и мастеров рассчитывается правильно.
- [ ] 4 графика Recharts рендерятся.
- [ ] Графики обновляются при смене салона/периода.
- [ ] Режим "Оба" — сравнительная таблица.
- [ ] При отсутствии данных — пустые графики с сообщением "Нет данных за период".
- [ ] Цвета графиков из палитры.
